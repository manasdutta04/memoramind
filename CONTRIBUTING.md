# Contributing to MemoraMind

First off, thank you for considering contributing to MemoraMind! It's people like you that make MemoraMind such a great tool.

## Where do I go from here?

If you've noticed a bug or have a feature request, please open an issue! It's better if you check first if there's already an open issue for it.

### Setting up the Project Locally

To run the project locally, please refer to the instructions in the [README.md](README.md). You will need Node.js and Python installed, as well as API keys for Mistral and ElevenLabs.

### Submitting a Pull Request

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints (run standard `.eslintrc` and `flake8`/`black` checks).
6. Issue that pull request!

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.
